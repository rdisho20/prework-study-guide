HTML

  • HTML Basics
    • Structure: <html>, <head>, <body>
    • Elements: Tags like <div>, <p>, <h1>, etc.
    • Attributes: Example: <a href="url">
  • Text Elements
    • <h1>, <h2>, ... for headings.
    • <p> for paragraphs.
    • <strong>, <em> for bold and italic text.
  • Links and Media
    • <a href="url"> for links.
    • <img src="src" alt="description"> for images.
  • Lists
    • <ul>, <ol>, <li> for unordered/ordered lists.
  • Semantic HTML
    • <header>, <footer>, <main> for document sections.
  • Meta Information
    • <meta charset="UTF-8"> for character encoding.
    • <title> for page title.
    • <meta name="description" content="..."> for page description.

CSS

  • The `*` selector applies styles globally (e.g., `margin: 0; padding: 0;`).
  • Multiple selectors can share styles using commas (e.g., `header, footer`).
  • DRY (Do Not Repeat Yourself) encourages reusing code for efficiency.
  • Text can be centered with `text-align: center`.
  • Images can use `display: block` and `margin: auto` for centering.
  • `ul` lists can have custom padding (`padding-left: 40px`) and font sizes.

"The" Box Model

CSS Box Model: Defines how elements render, encompassing content, padding, border, and margin.

Key Concepts:

  • Box Types:
    • Block: Starts on new line, respects dimensions, affects spacing w/ padding, borders, and margins.
    • Inline: Flows in-line, ignores height/width, affects horizontal spacing.
  • Box Layers:
    • Content Box: For content (`width`, `height`).
    • Padding Box: Space around content (`padding`).
    • Border Box: Around padding (`border`).
    • Margin Box: Space between elements (`margin`).
  • Box Models:
    • Standard: Width/height = content only; padding/borders add → total size.
    • Alternate: Width/height = content + padding + borders (`box-sizing: border-box`).

Git

  • Git Workflow
    • Branching: Create feature branches for isolated work.
    • Commit: Regular, descriptive commits on local branches.
    • Push: Push changes to remote after local commits.
    • Pull: `git pull origin main` to sync with remote.
    • Merge: Merge feature branches via PR.
  • Common Commands
    • `git clone `: Clone remote repository.
    • `git add .`: Stage changes.
    • `git commit -m "message"`: Commit staged changes.
    • `git push origin branch`: Push local branch to remote.
    • `git pull origin main`: Sync local with remote after PR merge.
    • `git status`: Check repository status.
  • Why Pull After Merging PR
    • Remote merge doesn’t auto-sync local branch.
    • `git pull` updates local to reflect remote changes.
    • Avoids conflicts in future work.
  • Syncing Local with Remote
    • Use `git pull origin main` to sync local after PR merge.
    • Alternative: `git fetch origin && git reset --hard origin/main` to avoid merge.

JavaScript

  • Basics
    • Variables: let, const, var
    • Data Types: String, Number, Boolean, Object, Array
    • Operators: +, -, *, /, %, ==, ===
  • Functions
    • Declaration: function name() {}
  • Loops
    • For: for (let i = 0; i < n; i++) {}
  • Conditionals
    • If: if (condition) {}
    • If-Else: if (condition) {} else {}
  • Objects & Arrays
    • Arrays: const arr = [1, 2, 3];
    • Access: arr[0]
  • Events
    • Event Listeners: element.addEventListener('event', callback);
    • Common events: click, submit, change
  • Miscellaneous
    • Console: console.log(), console.error()
